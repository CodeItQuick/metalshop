authoritarian-client

rule authentication and authorization with an iron fist

you're now looking at authoritarian-client's ui web components demo

authoritarian's auth-server microservice provides auth for these components, and also for other microservices

it's all a work in progress, but ultimately i want to create more easy-to-install microservices and components to cover use-cases like these:

  1. user auth, to facilitate login and logout for your website or app via google, passes tokens around and stuff
  2. paywalls and premium users, subscription payments via integration with braintree
  3. restricted content, for those aforementioned sweet premium users
  4. comments and questions, for user interactions
  5. simple integrations with your own features, use authoritarian npm package to verify tokens and check user claims in your own microservices
